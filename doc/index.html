    rapid-render

    Rapid

    一个无状态、数据驱动的高性能 WebGL 2D 游戏渲染引擎。
    A stateless, data-driven, high-performance WebGL 2D game rendering engine.

    ✨ 简介 / Introduction

    Rapid 不是一个"游戏引擎"，它是一个渲染库。它不管理场景树、不持有游戏对象状态、不内置物理——它只做一件事：极其高效地把东西画到屏幕上。

    你负责游戏逻辑，Rapid 负责渲染。这使得它极易与任何架构集成，不管你是 ECS、OOP 还是纯函数式。

    Rapid is not a "game engine" — it is a rendering library. It manages no scene graph, holds no game object state, and has no built-in physics. It does exactly one thing: draw things to the screen with extreme efficiency.

    You own your game logic. Rapid owns the pixels.

    🏗️ 架构概览 / Architecture Overview

    Rapid (主实例 / Main Instance)
    ├── MatrixStack        ← 层级变换栈 / Hierarchical transform stack
    │   └── MatrixStore    ← 扁平化矩阵池（Float32Array）/ Flat matrix pool
    ├── TextureManager     ← 纹理加载与缓存 / Texture loading & caching
    ├── SpriteRegion       ← 实例化批量精灵渲染 / Instanced batch sprite rendering
    └── GraphicRegion      ← 任意几何图形渲染 / Arbitrary geometry rendering

    核心设计理念 / Core design principles:

    原则 说明
    无状态渲染 引擎不存储"当前位置/旋转/缩放"，变换由你每帧构造并传入
    自动批处理 连续的精灵绘制调用自动合并为单次 GPU instanced draw call
    零分配热路径 MatrixStore 使用预分配的 Float32Array，每帧 reset 复用
    数据驱动 矩阵以整数 index 引用，便于任意架构集成

    🚀 快速开始 / Quick Start

    安装 / Installation

    目前为源码集成方式，直接引入 src/main.ts。

    import { Rapid } from './src/main';
    import { Color } from './src/color';

    初始化 / Initialization

    const canvas = document.getElementById('canvas') as HTMLCanvasElement;

    const rapid = new Rapid({
        canvas,
        logicWidth: 640,    // 逻辑宽度（可与物理像素不同）
        logicHeight: 480,   // 逻辑高度
        physicsWidth: 640,  // 物理像素宽度（可选，默认 CSS 尺寸 × DPR）
        physicsHeight: 480,
        backgroundColor: new Color(0.05, 0.05, 0.12, 1), // RGBA 0-1
    });

    最小游戏循环 / Minimal Game Loop

    const texture = await rapid.texture.load('./sprite.png');

    function loop(dt: number) {
        rapid.clear();

        // 设置变换 / Set transform
        rapid.matrixStack.identity();
        rapid.matrixStack.translate(100, 100);

        // 绘制精灵 / Draw sprite
        rapid.drawSprite(texture);

        rapid.flush();
        requestAnimationFrame((t) => loop(t / 1000));
    }

    requestAnimationFrame((t) => loop(t / 1000));

    📐 MatrixStack — 层级变换 / Hierarchical Transforms

    MatrixStack 是 Rapid 中最核心的概念。它模仿 Canvas 2D 的 save/restore API，但底层是高度优化的扁平化矩阵池（MatrixStore），所有矩阵以 6 个 float 存储（2D 仿射矩阵：a, b, c, d, tx, ty）。

    基本用法 / Basic Usage

    const ms = rapid.matrixStack;

    ms.identity();           // 重置当前变换为单位矩阵
    ms.translate(100, 200);  // 平移
    ms.scale(2, 2);          // 缩放
    ms.rotate(Math.PI / 4);  // 旋转（弧度）
    ms.rotateWithOffset(     // 绕指定锚点旋转（例如精灵中心）
        Math.PI / 4,
        tex.width / 2,
        tex.height / 2
    );

    rapid.drawSprite(texture);

    层级变换 / Hierarchical Transforms (Save / Restore)

    ms.identity();
    ms.save();                  // 保存当前变换状态（类似 ctx.save）
        ms.translate(300, 140);
        ms.rotateWithOffset(time, tex.width / 2, tex.height / 2);
        rapid.drawSprite(tex, Color.Red());     // 父节点

        ms.translate(50, 50);
        ms.save();              // 子节点继承父节点变换
            ms.scale(Math.sin(time) / 2 + 1, Math.sin(time) / 2 + 1);
            rapid.drawSprite(tex, Color.Cyan()); // 子节点
        ms.restore();

    ms.restore();

    延迟更新（高级）/ Deferred Matrix Update (Advanced)

    此功能允许你先构造完整的层级结构，再修改局部矩阵，最后统一更新世界矩阵。这对于需要在绘制前根据父节点状态调整子节点变换的场景特别有用。

    // 1. 构造层级（save 返回矩阵索引）
    ms.identity();
    const stick1 = ms.save();       // 获取 stick1 的矩阵句柄
    ms.translate(200, 200);
    ms.scale(1.3, 1.3);

    const stick2 = ms.save();       // 获取 stick2（stick1 的子节点）
    ms.translate(128, 0);

    // 2. 直接修改局部矩阵（绕轴心旋转）
    rapid.matrix.rotateWithOffset(stick1.local, time, 0, texStick.height / 2);
    rapid.matrix.rotateWithOffset(stick2.local, time * 4, 0, texStick.height / 2);

    // 3. 统一更新 stick1 及其所有子节点的世界矩阵
    ms.updateMatrix(stick1);

    // 4. 使用指定的世界矩阵渲染
    rapid.drawSprite(texStick, Color.White(), false, false, undefined, stick1.world);
    rapid.drawSprite(texStick, Color.White(), false, false, undefined, stick2.world);

    坐标转换 / Coordinate Conversion

    // 局部坐标 → 世界坐标
    const worldPos = ms.localToWorld(0, 0);

    // 世界坐标 → 局部坐标（用于碰撞检测）
    const localPos = ms.worldToLocal(mouseX, mouseY);

    // 获取当前世界变换信息
    const pos = ms.getGlobalPosition();   // { x, y }
    const scale = ms.getGlobalScale();    // { x, y }
    const rot = ms.getGlobalRotation();   // number (radians)

    🎨 图形绘制 / Graphic Drawing

    GraphicRegion 允许你绘制任意几何体（三角形、圆形、多边形、遮罩形状等）。

    // 绘制矩形
    rapid.startGraphic(rapid.gl.TRIANGLE_FAN);
    rapid.addRectVertex(200, 100, new Color(255, 0, 0, 200));
    rapid.endGraphic();

    // 绘制圆形
    rapid.startGraphic(rapid.gl.TRIANGLE_FAN);
    rapid.addCircleVertex(50, new Color(0, 255, 0, 200), 64); // r=50, 64段
    rapid.endGraphic();

    // 绘制带纹理的几何体
    rapid.startGraphic(rapid.gl.TRIANGLE_FAN, texture);
    rapid.addRectVertex(texture.width, texture.height);
    rapid.endGraphic();

    // 手动添加顶点
    rapid.startGraphic(rapid.gl.TRIANGLES);
    rapid.addGraphicVertex(0,   0,   0, 0, 0xFF0000FF);  // x, y, u, v, color
    rapid.addGraphicVertex(100, 0,   1, 0, 0xFF00FF00);
    rapid.addGraphicVertex(50,  100, 0.5, 1, 0xFFFF0000);
    rapid.endGraphic();

    🎭 遮罩系统 / Mask System

    基于 WebGL Stencil Buffer 实现，支持内/外裁剪。

    // 1. 绘制遮罩形状（写入模板缓冲，不显示颜色）
    rapid.startDrawMask(1, 0xFF);
    rapid.drawMaskImage(maskTexture);            // 用纹理作为遮罩形状
    // 或者自定义几何形状：
    // rapid.startMaskGraphic(rapid.gl.TRIANGLE_FAN);
    // rapid.addCircleVertex(100);
    // rapid.endGraphic();
    rapid.endDrawMask();

    // 2. 进入遮罩区域渲染（EQUAL = 在遮罩内，NOT_EQUAL = 在遮罩外）
    rapid.enterMask(MaskType.EQUAL, 1, 0xFF);
    rapid.drawSprite(contentTexture);
    rapid.exitMask();

    // 3. 清除遮罩
    rapid.clearMask();

    🔆 混合模式 / Blend Modes

    import { BlendMode } from './src/main';

    rapid.setBlendMode(BlendMode.NORMAL);    // 标准 Alpha 混合（默认）
    rapid.setBlendMode(BlendMode.ADD);       // 叠加（发光效果）
    rapid.setBlendMode(BlendMode.MULTIPLY);  // 正片叠底（阴影）
    rapid.setBlendMode(BlendMode.SCREEN);    // 滤色（柔和发光）
    rapid.setBlendMode(BlendMode.ERASE);     // 擦除（减去 Alpha）

    注意：setBlendMode 会自动 flush 当前批次，应尽量减少切换频率。

    🖥️ 渲染纹理 / Render Texture (FBO)

    将场景渲染到离屏纹理，再作为普通纹理使用。

    const rt = rapid.texture.createRenderTexture(256, 256);

    // 每帧渲染到 rt
    rapid.enterRenderTexture(rt);
        rapid.clear();  // 可选：清除 rt
        rapid.drawSprite(someTexture);
        // ... 任何 rapid 绘制调用
    rapid.leaveRenderTexture();

    // 将 rt 作为纹理绘制到屏幕
    rapid.matrixStack.translate(0, 0);
    rapid.drawSprite(rt);

    ✏️ 自定义着色器 / Custom Shaders

    CustomGlShader 允许你通过注入片段（无需编写完整 GLSL）来扩展内置着色器。

    着色器注入 API

    import { CustomGlShader } from './src/webgl/glshader';

    const myShader = new CustomGlShader(
        // 顶点着色器注入（可选修改 position 和 region）
        `
    uniform float u_time;
    void vertex(inout vec4 position, inout vec2 region) {
        // 让顶点随时间抖动
        position.x += sin(u_time + region.y * 6.28) * 10.0;
    }`,
        // 片段着色器注入（修改最终颜色）
        `
    uniform float u_time;
    uniform sampler2D u_extra_tex;
    void fragment(inout vec4 color) {
        color.rgb *= 0.7;
        color.rgb += texture(u_extra_tex, vRegion).rgb * 0.3;
        color.g += sin(u_time) * 0.1;
    }`,
        1  // 此 shader 额外占用的纹理单元数（如 u_extra_tex 占 1 个）
    );

    // 设置 uniform
    myShader.setUniforms({
        'u_time': 0.0,
        'u_extra_tex': overlayTexture.glTexture!,
    });

    // 每帧更新 uniform
    myShader.setUniforms({ 'u_time': time });

    // 绘制时传入
    rapid.drawSprite(texture, undefined, false, false, myShader);

    🎛️ API 速查 / API Reference

    Rapid

    方法 / 属性 说明
    rapid.clear() 清屏并重置 drawcallCount、matrixStack
    rapid.flush() 强制提交当前批次到 GPU
    rapid.drawSprite(tex, color?, flipX?, flipY?, shader?, matrixIdx?) 绘制精灵
    rapid.startGraphic(mode?, tex?, shader?) 开始图形绘制
    rapid.addGraphicVertex(x, y, u, v, color) 添加图形顶点
    rapid.addRectVertex(w, h, color?) 添加矩形顶点（4顶点）
    rapid.addCircleVertex(r, color?, segments?) 添加圆形顶点
    rapid.endGraphic() 结束图形绘制并提交
    rapid.enterRenderTexture(rt) 开始渲染到 FBO
    rapid.leaveRenderTexture() 结束 FBO 渲染
    rapid.startDrawMask(ref?, mask?) 开始写入遮罩
    rapid.endDrawMask() 结束写入遮罩
    rapid.enterMask(type, ref?, mask?) 进入遮罩渲染（EQUAL / NOT_EQUAL）
    rapid.exitMask() 退出遮罩渲染
    rapid.clearMask() 清除遮罩
    rapid.setBlendMode(mode) 设置混合模式
    rapid.resize(lW, lH, pW?, pH?) 调整画布与投影
    rapid.drawcallCount 当前帧 draw call 次数（调试用）

    MatrixStack

    方法 说明
    ms.identity() 重置为单位矩阵
    ms.translate(x, y) 平移
    ms.scale(sx, sy) 缩放
    ms.rotate(radians) 旋转
    ms.rotateWithOffset(r, ox, oy) 绕偏移点旋转（锚点旋转）
    ms.save() 保存状态，返回 { world, local, step }
    ms.restore() 恢复上一个状态
    ms.updateMatrix(step) 延迟更新指定节点及子节点世界矩阵
    ms.localToWorld(x, y) 局部 → 世界坐标
    ms.worldToLocal(x, y) 世界 → 局部坐标（碰撞检测）
    ms.getGlobalPosition() 获取当前世界坐标 {x, y}
    ms.getGlobalScale() 获取当前世界缩放 {x, y}
    ms.getGlobalRotation() 获取当前世界旋转（弧度）

    ⚡ 性能说明 / Performance Notes

    • 精灵批处理：连续的 drawSprite 调用（相同着色器、未超出纹理单元上限）会自动合并为单次 instanced draw call。切换着色器、混合模式或 FBO 会触发 flush。
    • MatrixStore：所有矩阵存储在单块 Float32Array，以整数 index 访问，避免对象分配，缓存友好。
    • 每帧 reset：rapid.clear() 调用 matrixStack.reset()，所有矩阵内存在帧间复用，无 GC 压力。
    • texSubImage2D：动态纹理（视频、Canvas）尺寸未变时走快速更新路径，避免重新分配 GPU 内存。

    🤝 设计哲学 / Design Philosophy

    Rapid 故意不提供以下功能：

    • ❌ 场景图 / 游戏对象树
    • ❌ 内置物理
    • ❌ 动画系统
    • ❌ 输入管理
    • ❌ 音频

    这些都是你的职责。Rapid 只是一块高性能画布，你用什么架构（ECS、OOP、纯函数、响应式）绘制它完全取决于你。

    这样设计的好处是：Rapid 的源代码极少，易于阅读、修改和嵌入任何项目。
